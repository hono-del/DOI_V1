```
✅ 戦略的優位性が一目瞭然
```

## 📐 新しい軸の定義

### X軸: レイヤー統合度（0-100）

**定義**: 機能レイヤーと対象レイヤーをどれだけ広くカバーしているか

```
0-30: 狭い統合
  - 単一機能（コンテンツ制作のみ、検索のみ）
  - 単一対象（Userのみ、OEMのみ）

30-60: 中程度の統合
  - 複数機能（制作+配信、ガイド+ナビ）
  - 複数対象（User+Dealer、Dealer+OEM）

60-100: 広範な統合
  - 全機能統合（制作〜体験OS）
  - 全対象統合（User〜OEM）
```

#### 計算方法

```
統合度 = (機能幅の正規化値 + 対象幅の正規化値) / 2

例：
- Paligo: (15 + 15) / 2 = 15  ← 狭い
- MSX: (45 + 65) / 2 = 55     ← 中程度
- 自社: (80 + 100) / 2 = 90   ← 最も広い
```

---

### Y軸: 価値創造志向（0-100）

**定義**: 業務最適化から体験創造へのスペクトラム

```
0-40: 業務最適化志向
  - 効率化、コスト削減、ROI重視
  - 既存プロセスの改善
  - 例: ServiceNow, MSX, Siemens

40-70: ハイブリッド
  - 業務最適化 + 一部の体験向上
  - 例: Helpfeel, Coveo, Tesla

70-100: 体験創造志向
  - 新しい体験の設計
  - ユーザー価値の創造
  - Usage Context理解
  - 例: 体験OS（自社）
```
