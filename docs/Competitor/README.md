## 📊 6次元分析フレームワーク

### 基本軸（2D配置）
- **X軸**: 機能レイヤー（10-90）
  - コンテンツ制作 → 構造化管理 → Digital Manual → 状況別ガイド → 行動ナビ → 体験OS
- **Y軸**: Lifecycle（20-90）
  - User(20) → Dealer(60) → OEM(90)

### 追加次元（視覚エンコーディング）
- **色**: 市場成熟度（レガシー → 成長期 → 新興 → 次世代）
- **パターン**: AI活用度（非AI → AI補助 → AI中心 → AI Native）
- **透明度**: 企業規模（中 → 大 → 特大）
- **フィルター**: 業界（自動車、製造業、カスタマーサポート、金融、全業界）← **NEW!**

## 🔍 重要な戦略的洞察

### 自社ポジション: Experience Infrastructure Provider

**タグライン**:  
"We don't build AI. We build the infrastructure that makes AI understand products."

### 3つの差別化要素

1. **Digital Manual-first Experience Design**
   - 困りごと起点のIntent/Journey設計
   - 取説ソースデータの構造化知見

2. **最も広範な垂直統合**
   - X軸: 40-90（Digital Manual〜体験OS）
   - Y軸: 20-90（User〜OEM）

3. **Experience Ontology as a Core Asset**
   - データ構造という模倣困難な資産
   - AI時代における真の競争優位性

### 主要競合との差別化

| 競合           | 脅威度 | 差別化ポイント                    |
|--------------|--------|------------------------------|
| ServiceNow   | 高      | 我々は製品情報起点、彼らはCRM起点     |
| Siemens      | 高      | 我々はUser起点、彼らはOEM起点      |
| Coveo        | 中-高   | 我々は構造化知見、彼らは検索起点      |
