# 超シンプルフロー入力形式

## コンセプト

**箇条書き形式で書くだけ。インデントでフローの構造を表現。**

---

## 入力例: `flow_simple.txt`

```
テーマ: スマートキーでドアが開かない
ID: THEME-001
緊急度: high

▼ ドアが開かない

  ? インジケーターは点灯する?
    → いいえ: キー電池切れ疑い
    → はい: 車両側問題疑い

■ キー電池切れ疑い
  
  ? どうしたい?
    → すぐ開けたい: 機械キー使用
    → 電池交換したい: 電池交換

  ■ 機械キー使用
    
    ? 機械キーはある?
      → はい: 機械キー取り出し
      → いいえ: 機械キー未所持
    
    ■ 機械キー取り出し
      [GUIDE] 機械キーの取り出し方
      
      ? 機械キーで開錠できた?
        → はい: ✓ 解決
        → いいえ: ✗ ドアロック故障
    
    ■ 機械キー未所持
      [GUIDE] 機械キーの場所・保管方法
      → ⚠ ロードサービス

  ■ 電池交換
    
    ? どうしたい?
      → 自分で交換: 電池交換準備
      → 販売店で交換: 販売店予約
    
    ■ 電池交換準備
      [GUIDE] 事前準備物（マイナスドライバー・小さいマイナスドライバー・CR2450電池）
      
      ? 準備物は揃ってる?
        → はい: 電池交換実施
        → いいえ: 電池購入サポート
    
    ■ 電池購入サポート
      [GUIDE] 電池購入方法（オンライン購入・近くの店舗検索）
      
      ? 電池を入手した?
        → はい: 電池交換実施
        → いいえ: 販売店予約
    
    ■ 電池交換実施
      [GUIDE] 電池交換手順（注意: 部品破損のおそれあり）
      
      ? 電池交換後、解決した?
        → はい: ✓ 解決
        → いいえ: 電池交換でも未解決
    
    ■ 販売店予約
      [GUIDE] 販売店予約・連絡先
      → ⚠ 販売店対応
    
    ■ 電池交換でも未解決
      
      ? 車両の電装品は動く?
        → いいえ: バッテリー上がり
        → はい: 電波干渉診断

■ 車両側問題疑い
  
  ? 車両の電装品は動く?
    → いいえ: バッテリー上がり
    → はい: 電波干渉診断

■ バッテリー上がり
  
  ? ジャンプできる?
    → はい: [GUIDE] ジャンプ手順
    → いいえ: ロードサービス

■ 電波干渉診断
  
  ? 空港や放送局の近く?
    → はい: 電波干渉対応
    → いいえ: その他診断
  
  ■ 電波干渉対応
    [GUIDE] キーを車体に近づける
    
    ? 解決した?
      → はい: ✓ 解決
      → いいえ: その他診断

■ その他診断
  
  ? 予備キーある?
    → はい: 代替キー使用
    → いいえ: ✗ ドアロック故障
  
  ■ 代替キー使用
    
    ? 予備キーで解決?
      → はい: 元のキー故障
      → いいえ: ✗ ドアロック故障

■ 元のキー故障
  [GUIDE] キー修理・交換
  → (終了)

✗ ドアロック故障
  [GUIDE] 故障時の対応
  → ロードサービス

⚠ ロードサービス
  [GUIDE] 連絡先と依頼方法
  → (終了)

✓ 解決
  [GUIDE] 予防策・定期点検
  → (終了)
```

---

## 記法ルール

### **記号の意味**

| 記号 | 意味 | 説明 |
|:-----|:-----|:-----|
| `▼` | 開始 | フローの起点 |
| `■` | ノード | 状態・処理 |
| `?` | 判定 | Yes/No、選択肢 |
| `→` | 遷移 | 次のノードへ |
| `[GUIDE]` | ガイド | 案内コンテンツ |
| `✓` | 解決 | 成功終了 |
| `✗` | 問題 | 故障・失敗 |
| `⚠` | 警告 | エスカレーション |

### **インデント**
- スペース2つでネスト
- 親子関係を表現

## さらに簡略化した形式

もっとシンプルにしたい場合は、矢印だけで書けます:

```
スマートキーでドアが開かない (THEME-001)

開始 → インジケーター点灯? 
  ├─ No → 電池切れ疑い → どうしたい?
  │         ├─ すぐ開けたい → 機械キー → 持ってる?
  │         │                    ├─ Yes → 開錠 → 開いた? 
  │         │                    │           ├─ Yes → 解決
  │         │                    │           └─ No → 故障
  │         │                    └─ No → 未所持 → サービス
  │         └─ 電池交換 → 交換後解決? 
  │                       ├─ Yes → 解決
  │                       └─ No → 車両診断 → ...
  └─ Yes → 車両側問題 → 車両診断 → ...
```

## もっとシンプルに: Q&A形式

質問ベースで書く方法:

```
テーマ: スマートキーでドアが開かない

Q1. インジケーターは点灯しますか?
  A1-1. いいえ → Q2へ
  A1-2. はい → Q10へ

Q2. どうしたいですか?
  A2-1. すぐ開けたい → Q3へ
  A2-2. 電池交換したい → Q6へ

Q3. 機械キーを持っていますか?
  A3-1. はい → Q4へ
  A3-2. いいえ → 【機械キー未所持】→ ロードサービス

Q4. 機械キーで開きましたか?
  A4-1. はい → 【解決】
  A4-2. いいえ → 【ドアロック故障】→ ロードサービス

Q6. 電池交換後、解決しましたか?
  A6-1. はい → 【解決】
  A6-2. いいえ → Q7へ

Q7. 車両の電装品は動作しますか?
  A7-1. いいえ → Q8へ (バッテリー上がり)
  A7-2. はい → Q11へ (電波干渉診断)

Q8. ジャンプスタートできますか?
  A8-1. はい → 【ジャンプ手順】
  A8-2. いいえ → ロードサービス

Q10. [車両側ルート] 車両の電装品は動作しますか?
  A10-1. いいえ → Q8へ
  A10-2. はい → Q11へ

Q11. 空港や放送局の近くですか?
  A11-1. はい → Q12へ
  A11-2. いいえ → Q13へ

Q12. キーを車体に近づけたら解決しましたか?
  A12-1. はい → 【解決】
  A12-2. いいえ → Q13へ

Q13. 予備のキーはありますか?
  A13-1. はい → Q14へ
  A13-2. いいえ → 【ドアロック故障】→ ロードサービス

Q14. 予備のキーで解決しましたか?
  A14-1. はい → 【元のキー故障】
  A14-2. いいえ → 【ドアロック故障】→ ロードサービス
```

## 比較: 3つの形式

### **形式1: 記号ベース** (推奨)
```
■ ノード名
  ? 質問?
    → 選択肢1: 次へ
    → 選択肢2: 別の次へ
```
**メリット**: 視覚的、構造が分かりやすい

---

### **形式2: ツリー形式**
```
開始 → 質問?
  ├─ 選択肢1 → 次
  └─ 選択肢2 → 別の次
```
**メリット**: 超シンプル、書きやすい

---

### **形式3: Q&A形式**
```
Q1. 質問?
  A1-1. 選択肢1 → Q2へ
  A1-2. 選択肢2 → Q3へ
```
**メリット**: 番号管理、参照しやすい

---
