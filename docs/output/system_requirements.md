## システム概要

取扱説明書DXは、従来の「読む取扱説明書」を、AIエージェントを活用した「先回りして最適化した情報を届ける仕組み」へ進化させるDX事業のシステム基盤である。本システムは、取扱説明書を単体のドキュメントからAIデータ資産へ転換し、ユーザー体験（AX: Assistant Experience）の高度化と、メーカーの商品企画・開発へのデータ還流による改善サイクルを実現する。

### 1. システムの目的

**解決する具体的な課題:**
- 取扱説明書は読まれない：従来の「読む」前提の説明書では、実際の利用率が極めて低い
- 機能の未活用：自動車機能の高度化により、ユーザーは多くの機能を十分に活用できていない
- 情報が配布物として固定化されている：SDV（ソフトウェア定義車両）の普及により、ソフトウェアアップデートに伴って機能や操作方法が変化し続けるため、情報も継続的に更新される仕組みが必要
- 問い合わせ増加によるコスト増大：顧客満足度の低下や、問い合わせ増加によるコスト増大のリスク

**目標とする成果:**
- 機能利用率：30% → 70%
- 問合せ削減率：30%削減
- 緊急時問合せ削減率：50%削減
- プロアクティブ案内利用率：40%以上
- 回答精度：90-95%
- レスポンス時間：1秒以下
- NPS：50以上

**期待されるビジネスインパクト:**
- 2026年度：2億円（PoC実施、市場検証・基盤構築）
- 2027年度：12億円（5社導入、損益分岐点到達）
- 2028年度：35億円（ROI 30%以上）
- 初期投資2.5億円で3年以内黒字化可能性

### 2. 主な機能

**コア機能（3-5個）:**
1. **RAGベースの検索型AI**: 9つのインテント分類に対応した高精度な情報検索・回答システム（回答精度90-95%、レスポンス1秒以下）
2. **プロアクティブ情報提供**: 緊急時自動案内、タイミングベース案内、状況認識による先回り案内
3. **データクリーニング・構造化**: PDFやドキュメントから不要な要素を除去し、取扱説明書特有の構造に基づいて構造化
4. **マルチチャネル展開**: 車載AIエージェント、スマホアプリ、コールセンターなど複数チャネルで展開
5. **データ還流システム**: ユーザーの利用データや行動ログを商品企画へフィードバック

**差別化機能（2-3個）:**
1. **DOI独自のRAG最適化技術**: セマンティック境界チャンキング、マルチベクトル戦略、ハイブリッド検索による検索精度15-20%向上、検索速度40%向上
2. **インテント別動的最適化**: 9つのインテント（検索型5つ＋プロアクティブ型4つ）に対応した最適化システム（インテント検出精度95%+）
3. **Safe × Smart × Delightの3軸体験設計**: 誤操作防止、文脈認識による最適提案、潜在機能の発見提案

**付加価値機能（1-2個）:**
1. **取扱データ編集のAI化**: 従来の手作業による編集から、AI支援による効率化（制作工数60%削減）
2. **多様化対応機能**: 高齢者・外国人・デジタルネイティブ向けカスタマイズ機能

### 4. 実現方法の概要

**主要なテクノロジー要素:**
- **RAG（Retrieval-Augmented Generation）システム**: 取扱説明書に特化したデータ構造最適化技術
- **LLM（Large Language Model）**: GPT-4 / Claude 3.5 / Gemini 1.5 Pro（マルチプロバイダー対応）
- **ベクトルデータベース**: セマンティック検索のためのベクトルDB
- **リアルタイムデータ処理**: 車両状態、環境データ、ユーザー行動のリアルタイム処理
- **プロアクティブ判定エンジン**: 状況データから自動で情報提供のタイミングを判定

**重要なインテグレーションポイント:**
- **車載システム連携**: 車載AIエージェント、車両データ（OBD-II、コネクテッドカーAPI）
- **外部サービス連携**: 天気情報、交通情報、地図データ、イベント情報
- **OEMシステム連携**: メーカーの既存システム、コールセンターシステム
- **モバイルアプリ連携**: スマホアプリ、プッシュ通知、AR機能

**スケーラビリティへの考慮:**
- 初期：50-100ユーザー（PoC段階）
- Phase 1：1,000-10,000ユーザー
- Phase 2：10,000-100,000ユーザー
- 最終目標：MAU 100万人
- マルチテナント対応：複数OEM、複数車種への対応
- クラウドネイティブアーキテクチャによる水平スケーリング

## 機能要件

### FR-1: RAGベースの検索型AI

**概要**: 9つのインテント分類に対応した高精度な情報検索・回答システム

**詳細要件:**
- **FR-1.1 データクリーニング・構造化**
  - PDFやドキュメントから不要な要素（ヘッダー・フッター・ページ番号など）を除去
  - 取扱説明書特有の構造（章・節・手順・注意事項など）に基づいて構造化
  - データ品質向上、誤回答削減

- **FR-1.2 セマンティック境界チャンキング**
  - 章・節・手順単位での分割
  - コンテキストオーバーラップ設計
  - 検索精度15-20%向上

- **FR-1.3 マルチベクトル戦略**
  - タイトル・本文・メタデータを個別にベクトル化
  - 用途に応じて使い分け
  - 検索速度40%向上

- **FR-1.4 ハイブリッド検索**
  - ベクトル検索+キーワード検索の統合（RRF: Reciprocal Rank Fusion）
  - 検索前フィルタリング（メタデータ活用で不要な検索を削減、コスト40%削減）

- **FR-1.5 インテント検出・ルーティング**
  - 9つのインテント分類（検索型5つ＋プロアクティブ型4つ）の検出
  - インテント検出精度95%+
  - インテント別に最適化された10種類以上のプロンプトテンプレート

- **FR-1.6 回答生成**
  - 回答精度90-95%
  - レスポンス時間1秒以下
  - 複合質問対応

**優先度**: 最高（コア機能）

---

### FR-2: プロアクティブ情報提供

**概要**: ユーザーが探さなくても、必要な情報が自動で届くシステム

**詳細要件:**
- **FR-2.1 緊急時自動案内**
  - 警告灯点灯時、異常検知時に自動で対応方法を提示
  - 緊急時問合せ削減率50%以上
  - 優先度最高の即座対応

- **FR-2.2 タイミングベース案内**
  - 納車直後、初回利用時など、特定タイミングで必要な情報を自動提示
  - 初期問合せの削減

- **FR-2.3 よくある質問の自動提示**
  - ユーザーの状況に応じて、関連するよくある質問を自動表示

- **FR-2.4 状況認識による先回り案内**（Phase 3以降）
  - 車両状態・環境（天候・道路状況など）を認識し、必要な情報を自動提示
  - 例：雨天時のワイパー操作、高速道路進入時のクルーズコントロール案内

- **FR-2.5 新機能・便利機能の発見提案**（Phase 4）
  - ユーザーが知らない便利な機能や使いこなし方を提案（Delight体験）
  - 機能発見率：20-30% → 60%以上

- **FR-2.6 環境・状況に応じた最適提案**（Phase 4）
  - ユーザーの知識レベル・使用履歴・環境を総合的に判断し、最適な情報を提示（Smart体験）

**優先度**: 最高（差別化機能）

## 技術スタック

### 推奨技術スタック（第一候補）

プロンプトの指示に従い、以下のモダンな技術スタックを第一候補として検討：

#### フロントエンド
- **Next.js (React)**: 
  - **選定理由**: 
    - サーバーサイドレンダリング（SSR）による高速な初期表示
    - API Routesによるフルスタック開発が可能
    - Vercelとの統合による簡単なデプロイ
    - 豊富なエコシステムとコミュニティ
    - SEO対応（管理画面、公開ページ用）

- **TypeScript**: 
  - **選定理由**: 
    - 型安全性によるバグの早期発見
    - 大規模開発での保守性向上
    - 開発効率の向上

#### バックエンド（BaaS）
- **Supabase**: 
  - **選定理由**: 
    - PostgreSQLベースのマネージドデータベース
    - 認証機能（Auth）の内蔵
    - リアルタイム機能（Realtime）の内蔵
    - RESTful APIの自動生成
    - オープンソース、セルフホスティング可能
    - コスト効率が良い（初期段階）

#### データベース
- **Neon (PostgreSQL)**: 
  - **選定理由**: 
    - サーバーレスPostgreSQL
    - 自動スケーリング
    - バックアップ・リカバリ機能
    - Supabaseと統合可能
    - コスト効率が良い

#### ホスティング
- **Vercel**: 
  - **選定理由**: 
    - Next.jsとの完璧な統合
    - 自動デプロイ、プレビュー環境
    - グローバルCDN
    - サーバーレス関数によるスケーラビリティ
    - 無料プランから開始可能

✅ **メリット:**

- **RAGシステム（バックエンドAPI）**: Python + FastAPI + LangChain
