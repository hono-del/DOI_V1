# トラブル対応優先度判定 - ユーザー情報提供・行動誘導設計

## 📋 概要

トラブル対応優先度判定マトリクスに基づき、各判定IDごとにユーザーに提供する情報と次の行動への誘導内容を「状態理解」「判断支援」「行動誘導」の3ステップで定義したドキュメントです。

---

## 🎯 設計方針：3ステップアプローチ

運転者への情報提供を以下の3ステップに構造化します：

### **ステップ1：状態理解**
運転者が現在の状況を正確に把握するための情報

**必須要素：**
- ✅ **緊急度表示**（【緊急度：最高/高/中/低】）
- ✅ **トラブル内容**（何が起きているか）
- ✅ **考えられる原因**（部品名・故障箇所）
- ✅ **付加的な状況**（走行中/停車中、同乗者情報等）

**目的：**
- 「今、何が起きているのか」を理解する
- 危険度を認識する
- 原因の可能性を知る

---

### **ステップ2：判断支援**
運転者が適切な判断を下すための情報

**必須要素：**
- ✅ **リスクの説明**（継続走行の危険性、影響範囲）
- ✅ **対応内容**（誰が、いつ、何をするか）
- ✅ **安心感の提供**（サポート体制の説明）

**目的：**
- 「どうすればいいのか」の方向性を理解する
- 対応の優先順位を判断できる
- 専門家のサポートがあることを認識する

---

### **ステップ3：行動誘導**
運転者が具体的なアクションを取るための指示

**必須要素：**
- ✅ **具体的なアクション**（ボタン操作、停車方法等）
- ✅ **注意事項**（やってはいけないこと）
- ✅ **補足ガイド**（追加の安全確保方法）

**目的：**
- 「今すぐ何をするか」を明確にする
- 迷わず行動できるようにする
- 安全を確保する

---

## 📊 3ステップの優先度別特徴

| 対応チャネル | 誘導方法 | ボタン表示 |
|------------|---------|-----------|
| **緊急ロードサービス** | ワンタップで即座に手配 | 「緊急対応」ボタン（赤色） |
| **ロードサービス** | 現在地自動送信で手配 | 「ロードサービス手配」ボタン（オレンジ色） |
| **コールセンター** | チェックリスト実施後に連絡 | 「症状チェック」→「コールセンター連絡」ボタン |
| **ディーラー予約** | 最寄りディーラー検索と予約 | 「ディーラー予約」ボタン（青色） |

### **最高優先度（スコア 1.5以上）- 3ステップの例**

#### **ステップ1：状態理解**
```
【緊急度：最高】
高速道路・雪・深夜で走行中にエンジンが停止しました。
お子様同乗・初心者ドライバーです。

燃料切れ、燃料ポンプ故障、またはエンジン制御システムの異常の可能性があります。
複合的な要因が考えられます。
```

#### **ステップ2：判断支援**
```
複合リスクが最大の状況です。
直ちにハザードランプを点け、110番通報します。
緊急ロードサービス、警察、会社、家族への連絡、救護、代替交通を即座に手配します。
お子様の安全を最優先します。
```

#### **ステップ3：行動誘導**
```
「最高優先緊急対応」ボタンを押してください。
自動的に110番・会社・家族に通報します。
救護と代替交通を手配します。
ガードレールの外側に避難し、お子様をしっかり保護してください。
```

#### **該当する判定ID**
- J003: 高速道路上でのエンジン停止
- J004: ブレーキ異常（高齢者・走行中）
- J005: 高速道路上のパンク
- J009: ブレーキ警告灯（妊婦運転・高速・業務）
- J025: 走行中の燃料漏れ
- J030: 山間部でのエンジン停止（妊婦運転・業務）
- J031: 深夜のパンク（子供同乗）
- J033: エアバッグ警告灯（乳幼児同乗・送迎中）
- J036: 複合リスク最大（高速・雪・深夜・子供・初心者）

```

```

### **低優先度（スコア 0.29以下）- 3ステップの例**


```
晴天のため緊急性は低いですが、雨天走行前に修理が必要です。

```

#### **ステップ3：行動誘導**
```
「ディーラー予約」で修理予約をしてください。
修理完了まで雨天走行は避けてください。
天気予報を確認してください。
```

#### **該当する判定ID**

## 📱 機能別の行動誘導詳細

### **1. 緊急停車ガイド**

#### **対象トラブル**
- 走行中のエンジン停止
- ブレーキ異常
- 走行中の燃料漏れ

#### **ガイド内容**
1. **ハザードランプ点灯** - アニメーション表示
2. **路肩への移動** - ステップバイステップ指示
3. **三角表示板設置** - 位置と方法の図解
4. **安全な場所への避難** - ガードレールの外側へ
5. **緊急連絡** - ワンタップで通報

### **2. 症状チェックリスト**

#### **対象トラブル**
- 警告灯点灯（エンジン、バッテリー、ABS）
- 異音（エンジン、ブレーキ）
- 異臭

#### **チェック項目例（エンジン警告灯）**
- [ ] 点灯は点滅していますか？（はい/いいえ）
- [ ] エンジンの振動はありますか？（はい/いいえ）
- [ ] 異音はありますか？（はい/いいえ）
- [ ] アクセルの反応は正常ですか？（はい/いいえ）
- [ ] 最終オイル交換はいつですか？（選択式）

### **3. ディーラー予約フロー**

#### **対象トラブル**
- 低〜中優先度のトラブル
- 翌日可・当日中の対応推奨トラブル

#### **予約フロー**
1. **最寄りディーラー検索** - GPS自動取得
2. **営業時間確認** - リアルタイム表示
3. **空き状況確認** - カレンダー表示
4. **予約確定** - ワンタップで完了
5. **リマインダー設定** - 自動通知

### **4. 代替交通手段の手配**

#### **対象トラブル**
- 最高優先度トラブル
- 妊婦運転、子供同乗のケース
- 業務移動中のケース

#### **手配内容**
- **タクシー手配** - 現在地へ配車
- **レンタカー手配** - 最寄り営業所の案内
- **公共交通案内** - 最寄り駅・バス停の案内
- **家族への連絡** - 自動メッセージ送信

## 🔄 継続的な改善

### **1. ユーザーフィードバックの収集**

各判定IDごとに、対応完了後にフィードバックを収集：
- 情報提供の分かりやすさ（5段階評価）
- 行動誘導の適切性（5段階評価）
- 対応時間の満足度（5段階評価）
- 自由記述コメント

### **2. データ分析による最適化**

- **情報提供の改善**: 理解しにくい表現の洗い出し
- **行動誘導の改善**: ボタン押下率の分析
- **対応時間の最適化**: 実際の対応時間との乖離分析
- **チャネル選択の最適化**: ユーザー満足度に基づく調整

### **3. A/Bテスト**

- 情報提供の表現パターン
- ボタンデザイン・配置
- 通知音・振動パターン
- ガイドコンテンツの形式

## 📊 効果測定指標（KPI）

| 指標 | 目標値 | 測定方法 |
|-----|-------|---------|
| **緊急対応ボタン押下率** | 95%以上 | 最高優先度トラブルでのボタン押下数 |
| **平均対応開始時間** | 3分以内 | トラブル検知から対応開始までの時間 |
| **ユーザー満足度** | 4.5/5.0以上 | 対応完了後のアンケート評価 |
| **情報理解度** | 90%以上 | 「分かりやすかった」と回答した割合 |
| **再発防止率** | 20%向上 | 同一トラブルの再発率の低下 |

## ✅ まとめ

この設計により、以下を実現します：

1. ✅ **状況に応じた最適な情報提供** - 優先度に応じた表現と内容
2. ✅ **明確な行動誘導** - ワンタップで次のアクションへ
3. ✅ **安全の最優先** - 最高優先度トラブルでの自動通報
4. ✅ **ユーザー体験の向上** - 不安軽減と適切な対応促進
5. ✅ **継続的な改善** - データに基づく最適化

これにより、ユーザーは安心してトラブル対応を進めることができ、システムは最適な対応チャネルへの誘導を実現します。
