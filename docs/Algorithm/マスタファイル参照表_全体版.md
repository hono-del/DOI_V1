### **Excel 2：トラブル対応優先度判定マトリクス**

**判定軸（11軸）：**

**外部環境軸（4軸）：**
1. 道路種別（1-1）× 補正値
2. 天気環境（1-2）× 補正値
3. 時間帯（1-2-2）× 補正値
4. 状況（2-2）× 補正値

**状況軸（3軸）：**
5. 同乗者（1-13）× 補正値
6. 移動目的（1-14）× 補正値
7. 移動距離（1-15）× 補正値

**ユーザー属性軸（4軸）：**
8. 年齢層（1-16）× 補正値
9. 運転頻度（1-17）× 補正値
10. 車両タイプ（1-18）× 補正値
11. トラブル履歴（1-19）× 補正値

**基準値：**
- トラブル種別（2-1）→ 基本緊急性スコア

**参照マスタ：**
- ユーザー属性マスタ（2-3）
- 対応チャネルマスタ（2-5）

**計算式：**
```
総合優先度スコア = 基本緊急性スコア × 
                   道路種別補正 × 天気補正 × 時間帯補正 × 状況補正 ×
                   同乗者補正 × 移動目的補正 × 移動距離補正 ×
                   年齢層補正 × 運転頻度補正 × 車両タイプ補正 × トラブル履歴補正
```

**出力：**
- 総合優先度スコア、優先度ランク、推奨対応時間、対応チャネル

---

### **Excel 3：情報提示レベル調整マトリクス**

**判定軸（6軸）：**
1. 購入ステータス（3-1）× 補正値
2. 機能使用履歴（3-2）× 補正値
3. 年齢層（1-16）× 補正値
4. 運転頻度（1-17）× 補正値
5. トラブル履歴（1-19）× 補正値
6. 車両タイプ（1-18）× 補正値

**参照マスタ：**
- 説明レベル別メッセージサンプル（3-4）
- 警告メッセージレベル別サンプル（3-5）

**計算式：**
```
総合理解度スコア = 購入ステータス補正値 +
                   機能使用履歴補正値 +
                   年齢層補正値 +
                   運転頻度補正値 +
                   トラブル履歴補正値 +
                   車両タイプ補正値
```

**出力：**
- 総合理解度スコア、説明レベル（6段階）、推奨フォーマット、文字量

## 🔢 各マトリクスのスコア計算式

### **Excel 1：機能推奨タイミング判定**

```
推奨スコア = 道路種別スコア × 
             季節スコア × 
             天気スコア × 
             時間帯スコア × 
             速度交通量スコア × 
             知識レベル補正 × 
             使用経験補正
```

**判定結果：**
- 推奨タイミング（今すぐ/機会があれば/不適切）
- 推奨機能（ACC、LKA、BSM等）
- 優先度ランク

---

### **Excel 2：トラブル対応優先度判定**

```
総合優先度スコア = 基本緊急性スコア × 
                   道路種別補正 × 
                   天気補正 × 
                   時間帯補正 × 
                   状況補正 ×
                   同乗者補正 × 
                   移動目的補正 × 
                   移動距離補正 × 
                   年齢層補正 × 
                   運転頻度補正 × 
                   車両タイプ補正 × 
                   トラブル履歴補正
```

**判定結果：**
- 優先度ランク（最高/高/中/低）
- 推奨対応時間（即時～翌日）
- 対応チャネル（緊急ロードサービス/ロードサービス/コールセンター）

---

```
                   運転頻度補正値 +
```

## 🎯 具体的な計算例

### **例1：トラブル対応優先度判定**

**シナリオ：高速道路でエンジン停止・妊婦運転・業務中・社用車**

**入力：**
- トラブル：走行中エンジン停止（TRB_002）→ 基本緊急性スコア: 1.0
- 道路：高速道路（ROAD_002）→ 補正: 1.3
- 天気：晴れ（WEATHER_001）→ 補正: 0.9
- 時間帯：日中（TIME_003）→ 補正: 0.9
- 状況：走行中（SIT_001）→ 補正: 1.3
- 同乗者：妊婦運転（PASS_005）→ 補正: 1.25
- 目的：業務中（PURPOSE_002）→ 補正: 1.2
- 距離：長距離（DIST_003）→ 補正: 1.1
- 年齢層：中年層（AGE_002）→ 補正: 1.0
- 運転頻度：毎日（FREQ_001）→ 補正: 1.1
- 車両タイプ：社用車（VEH_TYPE_005）→ 補正: 1.25
- トラブル履歴：通常履歴（HIST_004）→ 補正: 1.0

**計算：**
```
総合優先度スコア = 1.0 × 1.3 × 0.9 × 0.9 × 1.3 × 1.25 × 1.2 × 1.1 × 1.0 × 1.1 × 1.25 × 1.0 
                = 2.91
```

**判定結果：**
- 優先度ランク：**最高**（スコア1.2以上）
- 対応時間：**即時～15分以内**
- 対応チャネル：**緊急ロードサービス**
- 備考：妊婦・業務車両・高速道路走行中で最優先対応

### **例2：情報提示レベル調整**

**シナリオ：新規購入直後・高齢者・ペーパードライバー・レンタカー・未使用機能**

**入力：**
- 購入ステータス：新規購入直後（STATUS_001）→ 補正値: +1.5
- 機能使用履歴：未使用（USAGE_001）→ 補正値: +1.5

- 車両タイプ：レンタカー（VEH_TYPE_003）→ 補正値: +0.75

**計算：**
```
総合理解度スコア = 1.5 + 1.5 + 0.3 + 0.35 + 0.0 + 0.75 
                = 4.4
```

**判定結果：**
- 説明レベル：**最詳細**（スコア3.0以上）
- 推奨フォーマット：**動画+ステップガイド+音声**
- 文字量：**300文字以上**
- 図解：**必須**
- 音声案内：**必須**
- ステップバイステップ：**必須**
- 専門用語使用：**最小限**
- 備考：車両に不慣れで最も詳細な説明が必要

### **例3：商談最適化**

**シナリオ：初めての車購入・夫婦+乳幼児・子供送迎・200-250万円・試乗後**

**入力：**
- 顧客プロファイル：初めての車購入（PROF_001）→ 補正値: +0.5
- 家族構成：夫婦+乳幼児（FAM_003）→ 補正値: +0.4
- 用途：子供送迎（USE_003）→ 補正値: +0.4
- 予算：200-250万円（BUD_002）→ 補正値: +0.3
- 商談シーン：試乗後（SCENE_002）→ 補正値: +0.6

**計算：**
```
総合提案レベル = 0.5 + 0.4 + 0.4 + 0.3 + 0.6
```

## ✅ マスタ管理のポイント

### **1. 共用マスタの管理**
- `excel_data/` フォルダのマスタは複数のマトリクスで共用
- 更新時は全マトリクスへの影響を確認
- 特に道路種別、天気、時間帯、年齢層、運転頻度、車両タイプは複数マトリクスで使用

### **2. 軸の追加・削除**
- 新しい軸を追加する場合は、マトリクスの計算式も更新
- 補正値の範囲：
  - 乗算型（Excel 1, 2）: 0.5～1.5を基本
  - 加算型（Excel 3, 4）: -1.0～+2.0を基本

### **3. バージョン管理**
- 重要な変更時はファイル名に`_v2`等のバージョン番号を追加
- 変更履歴を設計書に記録
- 現在の最新版：
  - 機能推奨ルールマトリクス：v4
  - トラブル対応優先度マトリクス：v6
  - 商談最適化マトリクス：v2

### **4. データの整合性**
- IDは必ず一意にする（例：ROAD_001、PROF_001）
- 補正値の合計が極端にならないよう注意
  - Excel 2: 総合スコアが10を超えないよう調整
  - Excel 3: 総合スコアが6.0を超えないよう調整
- テストケースで動作確認（2-6、その他各マトリクス）

### **5. 各マトリクスの独立性と連携**
- Excel 1（機能推奨）と Excel 3（情報提示）は連携
  - 推奨機能 → その機能の説明レベルを決定
- Excel 2（トラブル対応）と Excel 6（コールセンター）は連携
  - トラブル優先度 → コールセンター対応の緊急度
- Excel 4（商談最適化）と Excel 5（対応履歴予測）は連携
  - 商談シーン → 次のフォローアクション予測
