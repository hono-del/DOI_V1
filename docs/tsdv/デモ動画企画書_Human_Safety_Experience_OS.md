# デモ動画企画書
# Human Safety Experience OS
# ～交通事故ゼロを、人の理解から支える～

**作成日**: 2025年1月7日  
**目的**: トヨタ様へのプレゼンテーション用デモ動画  
**対象**: トヨタ自動車 SDV開発部門・商品企画部門  
**動画尺**: 5-7分

---

## 🎯 動画の目的

### 主目的
トヨタ様のSDV戦略「交通事故ゼロ」実現に、弊社のHuman Safety Experience OSが**どのように具体的に貢献するか**を視覚的に示す。

### 視聴後に持って帰ってもらいたい印象
1. **「これは必要だ」** - SDVの最後の壁を解決する唯一のソリューション
2. **「実現可能だ」** - 技術的に確立されており、すぐに始められる
3. **「トヨタに合う」** - トヨタのSDV戦略と完全に整合している

### 【第1章】問題提起：事故が起きる4つの層 (1分)

#### 映像構成

**シーン1: 認知の問題 (15秒)**
```
【映像】
- 警告灯が点灯
- ドライバー「？？？」（理解できない表情）
- 複数の警告灯一覧が表示
- ドライバーが取説を探すが見つからない

【テロップ】
「❌ 認知の問題」
「警告の意味を理解していない」

【データ表示】
「警告灯の理解率: わずか35%」
```

**シーン2: 判断の問題 (15秒)**
```
【映像】
- 家族でドライブ中（週末の郊外）
- 突然「ピーピー」と警告音
- 「ブレーキパッドの摩耗」警告灯が点灯
- ドライバー（父親）が困惑
  「え？今すぐ止まるべき？」
  「このまま家まで帰れる？」
  「子供乗ってるけど大丈夫？」
- 後部座席の子供「パパ、どうしたの？」
- 妻「どうする？」と不安そうな表情
- スマホで検索しようとするが運転中
- 時間が経過（5秒...10秒...）

【テロップ】
「❌ 判断の問題」
「今すぐ止まるべき？このまま帰れる？」

【データ表示】
「判断に迷う時間: 平均45秒」
「この間、危険な状態が続く」
```

**シーン3: 行動の問題 (15秒)**
```
【映像】
- 信号待ち中（運転席の視点）
- ドライバー（30代男性）がスマホ片手に
  「Bluetoothで音楽を聴きたい...」
- センターディスプレイをタッチ
  → 複雑なメニュー画面（設定項目が多数）
  → 「設定」→「接続」→「Bluetooth」...
- 迷いながらタッチしていく
- その時、信号が青に変わる
- 後ろから「プップー」とクラクション
- 焦って「キャンセル」を押そうとして
- 間違えて「初期化」ボタンをタップ！
- 「全設定を初期化しますか？」の確認画面
- ドライバー「え！？違う違う！」
- さらに焦って操作
- 画面がフリーズ気味に...
- 「やばい、どうしよう...」

別カット:
- ドライブスルーで注文中
- モバイルオーダーアプリを車内ディスプレイで操作
- 「クーポン適用」を押したつもりが
- 「注文キャンセル」を押してしまう
- 「あれ？消えた？」
- 後ろに車が並んでいる...焦り

【テロップ】
「❌ 行動の問題」
「複雑な操作で迷う・間違える」

【データ表示】
「操作ミスでの混乱: 65%が経験」
「運転中の誤操作: 事故リスク3倍」
```

**シーン4: 継続の問題 (15秒)**
```
【映像】
- 時間経過のアニメーション
- 同じ警告灯が再度点灯
- また同じように戸惑うドライバー
- 「前も見たけど忘れた...」

【テロップ】
「❌ 継続の問題」
「学習が定着しない」

【データ表示】
「安全機能の利用率: わずか10%」
```

#### ナレーション
```
「事故が起きる4つの層。

 第一に、認知の問題。
 警告灯や機能の意味を、ドライバーは理解していません。
 
 第二に、判断の問題。
 複数の警告が重なった時、何を優先すべきか瞬時に分かりません。
 
 第三に、行動の問題。
 正しい操作方法を迷い、誤操作してしまいます。
 
 第四に、継続の問題。
 一度学んでも、時間が経つと忘れてしまいます。
 
 これらは、ハードウェアでもAIでも解決できない、
 『情報体験の問題』なのです。」
```

### 全文（約1,800文字）

```
【オープニング】
トヨタ様が目指すSDV戦略、「交通事故ゼロ」の実現。

しかし、どれだけ車両側の安全技術が進化しても、
事故がなくならない"最後の壁"があります。

それは、「人間の理解・判断・行動」の問題です。

【第1章】
事故が起きる4つの層。

第一に、認知の問題。
警告灯や機能の意味を、ドライバーは理解していません。

第二に、判断の問題。
複数の警告が重なった時、何を優先すべきか瞬時に分かりません。

第三に、行動の問題。
正しい操作方法を迷い、誤操作してしまいます。

第四に、継続の問題。
一度学んでも、時間が経つと忘れてしまいます。

これらは、ハードウェアでもAIでも解決できない、
「情報体験の問題」なのです。

【第2章】
私たちが提案するのは、
「Human Safety Experience OS」。

人とSDVの間の"意味変換レイヤー"です。

OEMが内製しづらく、やると重くなる領域。
しかし、交通事故ゼロには不可欠な領域です。

このシステムは、3つのレイヤーで構成されています。

レイヤー1、Safety Knowledge Skeleton。
取扱説明書を「人の判断単位」で再構造化します。

単なる情報の羅列ではなく、
「この警告は何か」
「今すぐすべきこと」
「なぜ重要か」
「放置するとどうなるか」
を明確に示します。

回答精度は90から95パーセント。
レスポンスは1秒以内です。

レイヤー2、Contextual Safety Guidance。
車両データ、環境データ、ユーザーデータを統合し、
「今、何をすべきか」に翻訳します。

これは単なるAIチャットではありません。
体験設計された判断支援システムです。

状況に応じて、優先順位を明示し、
理由を納得できる形で説明します。

レイヤー3、Safety Experience Analytics。
認知、判断、行動、継続の4つの層で、
人間側の問題を可視化します。

このデータをOEMの商品企画にフィードバックすることで、
SDV開発の精度が向上します。

【第3章】
具体的なユースケースをご覧ください。

緊急時対応。
高速道路でパンクが発生しました。

システムは即座にパンクを検知し、
音声で段階的な対応手順をガイドします。

対応時間は、45秒から15秒に短縮。
二次事故のリスクを80パーセント削減します。

判断支援。
家族でドライブ中、ブレーキパッド摩耗の警告が出ました。

「今すぐ止まるべき？このまま帰れる？」

システムは状況を判断し、
「自宅まで帰れます。ただし速度控えめに」
と具体的にガイドします。

家族の不安を解消し、
適切な判断を支援します。

誤操作防止。
新車で初めてのドライブ。
住宅街で間違えてパワーモードを押しかけました。

システムが「今は適していません」と警告し、
エコモードをおすすめします。

誤操作を未然に防ぎ、
適切な機能選択を支援します。

学習定着。
デフロスターの使い方を、3回の体験で習得します。

初回は詳しく、2回目は簡潔に、
3回目には自力で操作。

同じことを何度も調べない。
自然に身につく設計です。

【第4章】
トヨタ様のSDV戦略との整合性。

トヨタ様は、車両側の安全を担当。
私たちは、人間側の安全を担当します。

この役割分担により、
交通事故ゼロの実現に近づきます。

私たちが提供する3つの価値。

第一に、説明責任の外部脳。
PL法・製造物責任に完全対応します。

第二に、判断支援の実現。
単なる情報提供ではなく、
体験設計された判断支援を提供します。

第三に、データ還流。
人間側のデータを商品企画にフィードバックし、
SDV開発の精度を向上させます。

【第5章】
期待される効果。

短期的には、コールセンターコストを30パーセント削減。
販売店の対応時間を40パーセント削減します。

中期的には、4つの層すべてで改善を実現。
事故率を5から10パーセント低減します。

長期的には、「最も安全なSDV」としての
ブランド確立を目指します。

【エンディング】
SDVで事故がなくならない"最後の壁"を、
Human Safety Experience OS で解決する。

"We translate SDV into human-safe experience."

事故ゼロ社会を、人の理解から支える。

まずは3ヶ月のPoCから始めませんか？

お問い合わせをお待ちしております。
```
